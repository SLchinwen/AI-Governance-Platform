# 技術規範問卷與 AI 協作治理平台 - 實施計畫

## 專案章程

### 專案名稱
**技術規範問卷與 AI 協作治理平台**
Tech Spec Questionnaire & AI Governance Platform

### 核心目標
建立一套公司內部可長期維護的技術規範問卷程式，使不同角色人員可依責任範圍填寫專案技術與治理資訊，系統再依據標準題庫、驗證規則與輸出模板，自動產出可供 AI 協作、文件治理與 GitHub 版本管理使用的正式資產。

### 預期成果
- 每個專案都有一致的技術定義流程
- 專案答案可轉為標準 JSON 與 Markdown
- AI 協作前可先做 readiness 評估
- 規格變更可追溯、可審核、可版本化
- 可持續累積成公司級技術知識庫

## 使用流程（六步驟）

### Step 1：建立新專案
PM 填寫專案名稱、類型、範圍等基本資訊

### Step 2：套用題庫
系統依專案類型自動載入對應題組（Web/API/整合/工具/維運）

### Step 3：角色分派填答
- **PM** → 需求與範圍（project_basic）
- **Architect** → 架構與設計模式（architecture, ai_collaboration）
- **Backend Lead** → API / DB / Auth / 測試（backend, database, api）
- **Frontend Lead** → UI / 框架 / 狀態管理（frontend）
- **DevOps** → 部署 / CI/CD / 監控（deployment）
- **Security Lead** → 資安 / 合規 / AI 邊界（security）

### Step 4：規則檢查
系統自動執行 20 條驗證規則，產出：
- 缺漏清單
- 衝突警告
- 低信心項目
- AI 協作風險提示

### Step 5：輸出正式資產
通過驗證後產出：
- `tech-stack.json` — 技術棧結構化資料
- `ai-context.md` — AI 開發上下文
- `readiness-report.md` — 就緒度評估報告

### Step 6：GitHub 發布
審核通過後發佈到正式 repo，透過 PR 流程管控。
