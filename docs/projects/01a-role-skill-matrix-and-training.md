# PRJ-001 角色 Skill 矩陣與補強計畫 v1.0

> 適用專案：PRJ-001 技術棧問券與 AI 協作知識庫系統  
> 目的：定義各角色最小可用能力，確保問券可有效填寫與決策。

---

## 1. 角色最小 Skill（MVP）

| 角色 | 必備 Skill（至少 3 項） | 目標成果 |
| ---- | ----------------------- | -------- |
| PM/顧問 | 需求摘要結構化、訪談重點轉決策項、AI 摘要驗證 | 可在 60 分鐘內產出可上傳 `md` |
| 架構師 | 技術選項取捨、風險判讀、AI 建議審核 | 可在規劃階段完成可審核決策 |
| 後端工程師 | API/資料模型對齊、選項化填答、錯誤回流記錄 | 可完成 S2-S4 問券與驗證 |
| 前端工程師 | 精靈式表單體驗、可用性文案、選項互動設計 | 填單操作 30 分鐘內完成 |
| DevOps/系統工程 | 流程 Gate 觀念、發布與回滾、稽核留痕 | 可完成 S5-S6 流轉與產出 |
| 資安 | AI 邊界審核、機敏資料分類、合規檢核 | 可判斷建議是否違反政策 |

---

## 2. 常見不足與對應補強

| 不足現象 | 風險 | 補強方式 |
| -------- | ---- | -------- |
| 填表偏自由文字 | 難標準化、AI 難學習 | 增加標準選項，文字改補充欄 |
| 技術選項看不懂 | 錯選或跳題 | 加入口語化提示與「何時選這個」 |
| AI 建議直接採納 | 決策品質不穩 | 強制「採納理由 + 影響評估」 |
| 問題未回流 | 知識庫不演進 | 設置回流單與週期審核 |

---

## 3. 2 週補強計畫（建議）

## 第 1 週：先補可用能力

- PM/顧問：需求摘要模板訓練（`md` 結構）
- 技術角色：決策題填寫演練（以真實案例）
- 全員：AI 建議審核規則演練（接受/拒絕/調整）

## 第 2 週：上線前實戰

- 跑一次完整 S1-S6 流程（30 分鐘目標）
- 盤點「最難決策題」前 10 名並補文案
- 建立第一批問題回流案例（至少 3 筆）

---

## 4. 上線判定（Skill 面）

- [ ] 每個角色至少完成 1 次模擬填單
- [ ] 每個角色至少完成 1 次 AI 建議審核
- [ ] PM/顧問可獨立產出需求摘要並觸發流程
- [ ] 技術角色能解釋其決策理由（非只選項）
