# PRJ-001 AI Skill 階段/領域引導手冊 v1.0

> 目的：在每個階段、每個領域提供 AI Skill 提醒與口語化決策引導，讓問券更容易填寫。  
> 核心原則：能選不寫、先提示再決策、決策需留痕。

---

## 1. AI Skill 設計原則

- **階段化**：每次只提示當前階段必要決策
- **領域化**：前端/後端/DB/API/部署/安全各有專屬提示
- **口語化**：以「什麼情況選這個」來說明選項
- **可審核**：AI 建議必附理由、風險、替代方案
- **可回流**：決策後可回寫「問題與解法」到知識庫

---

## 2. 階段 AI Skill 任務

| 階段 | AI Skill 任務 | 輸出 |
| ---- | ------------- | ---- |
| S1 需求收集 | 萃取 `md` 需求重點、列待決策清單 | 需求重點摘要、待決策項 |
| S2 載入題目 | 依角色與階段推薦應填題 | 角色題目清單、必填提醒 |
| S3 填答 | 對每個難題給口語化選項解釋 | 選項說明、建議值 |
| S4 驗證 | 解釋錯誤原因與修正順序 | 修正建議清單 |
| S5 審核 | 提供採納/拒絕建議與風險說明 | 審核備註草稿 |
| S6 產出 | 檢查輸出可讀性與一致性 | 產出品質檢查摘要 |

## 3. 領域 AI Skill（口語化範例）

## 3.1 後端領域

- 難決策題：`.NET 版本`
- 口語化提示：
  - 「如果團隊已有 .NET 8 維運經驗，先選 .NET 8 比較穩定。」
  - 「若要用新語法，需同時確認部署環境支援。」

## 3.2 前端領域

- 難決策題：`UI 元件庫`
- 口語化提示：
  - 「重視快速交付可選 Ant Design。」
  - 「重視客製化與一致風格可選 shadcn/ui。」

## 3.3 資料庫領域

- 難決策題：`Azure SQL DB vs 其他`
- 口語化提示：
  - 「若本案主力在 Azure，先選 Azure SQL 可降低維運複雜度。」
  - 「若有跨雲需求，再評估其他方案。」

## 3.4 安全領域

- 難決策題：`AI 可用資料範圍`
- 口語化提示：
  - 「含個資或機敏資料時，先選『禁止上傳原文』。」
  - 「必要時僅上傳去識別化摘要。」

## 4. 每題 AI 建議格式（固定）

每次建議需包含：

1. 建議選項
2. 建議理由（1-2 句）
3. 主要風險（若選此項）
4. 替代選項（至少 1 個）
5. 建議信心度（高/中/低）

---

## 5. 不好決策選項的提醒規則

- 若某題被改動超過 2 次 -> 顯示「建議開 15 分鐘決策會議」
- 若跨角色衝突 -> 顯示「需架構師 + 安全共同確認」
- 若 AI 信心度為低 -> 強制要求人工補充理由

---

## 6. 回饋閉環制度（AI 協助優化知識庫）

每次專案若出現問題：

1. 在問券標記「問題類型」
2. 填寫「實際解法」
3. AI 產生「是否要更新標準選項/規則」建議
4. 治理審核後決定：
   - 新增問題與解法條目
   - 更新題目選項
   - 更新驗證規則
   - 升版技術棧標準

---

## 7. 實作建議（系統）

- 問券每題加「AI 幫我說明」按鈕
- 問券每題加「我不確定」按鈕，觸發口語化引導
- 審核頁加「採納 AI 建議」與「拒絕並說明」兩鍵
- 後台加「AI 建議命中率」與「採納率」儀表板

---

## 8. 與標準 Prompt 的關聯

- 前置整理採用：`../prompts/prompt-calibration-v2.md`
- 品質自檢採用：`../prompts/prompt-qc.md`
- 未通過自檢時，僅可輸出「待確認清單」，不可直接推進正式決策
