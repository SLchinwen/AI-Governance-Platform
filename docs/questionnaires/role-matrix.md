## 三、各角色的填答指引

### PM — 專案經理

**你負責的題目：** 專案基本資訊（8 題）

**填答重點：**
- 確保專案名稱、類型、範圍的準確性
- 資料敏感度等級需與客戶/法務確認
- 時程與團隊規模需與管理層對齊
- 這些答案是後續所有題組的基礎

**常見錯誤：**
- 將專案類型選錯（如把 API 平台選成 Web 應用）
- 低估資料敏感度
- 未確認就回答

---

### Architect — 架構師

**你負責的題目：** 系統架構（8 題）+ AI 協作規範（9 題）

**填答重點：**
- 架構風格是最重要的決策，會影響所有後續產出
- 分層策略需與團隊對齊
- AI 協作的允許/禁止範圍是安全紅線
- 所有 AI 審核流程由你把關

**常見錯誤：**
- 選了微服務但實際上不需要
- AI 安全邊界定義過於寬鬆
- 未考慮既有系統的限制

---

### Backend Lead — 後端主管

**你負責的題目：** 後端（9 題）+ 資料庫（6 題）+ API（7 題）

**填答重點：**
- .NET 版本 / 框架類型直接影響 AI 產出
- ORM 選擇決定資料存取模式
- 認證方式需與架構師對齊
- API 回應格式是前後端協作的核心

**常見錯誤：**
- 忘記填 JWT 設定細節
- 資料表命名規範與既有系統不一致
- 未定義錯誤處理策略

---

### Frontend Lead — 前端主管

**你負責的題目：** 前端技術棧（9 題）

**填答重點：**
- 框架選型需考慮團隊熟悉度
- TypeScript strict 模式會影響 AI 產出品質
- UI 元件庫決定 AI 生成的 import 與用法
- 狀態管理方案需與專案複雜度匹配

**常見錯誤：**
- 建構工具與框架不匹配
- 選了元件庫但未統一版本
- 未定義表單處理方案

---

### DevOps — 維運工程師

**你負責的題目：** 部署與維運（8 題）

**填答重點：**
- CI/CD 平台選型影響所有 workflow 產出
- 環境清單需完整（尤其 staging/production）
- 監控與日誌是長期維運的基礎
- Git 分支策略影響團隊協作方式

**常見錯誤：**
- 無 CI/CD 但未說明原因
- 環境清單不完整
- 無集中式日誌平台

---

### Security Lead — 資安主管

**你負責的題目：** 資安與合規（6 題）

**填答重點：**
- 授權機制是最容易被忽略的安全環節
- 合規要求影響整個專案的安全設計
- 機敏資訊管理方式是安全底線
- AI 安全邊界需與架構師共同定義

**常見錯誤：**
- 低估合規要求
- 機敏資訊仍用環境變數管理
- 未定義 AI 安全邊界

## 四、跨角色協作點

某些題目需要多角色協同填答或確認：

| 協作場景 | 涉及角色 | 協作方式 |
|---------|---------|---------|
| 認證方式選型 | Backend + Architect + Security | 技術會議討論 |
| 部署平台選型 | Architect + DevOps | 技術方案評估 |
| API 回應格式 | Backend + Frontend | 前後端對齊會議 |
| AI 安全邊界 | Architect + Security | 安全審查會議 |
| 資料敏感度 | PM + Security | 合規評估 |
| 測試策略 | Backend + Frontend + QA | 品質規劃會議 |
