# 審核政策 v1.0

> 定義問卷答案審核、文件審核與 AI 產出審核的標準流程。

---

## 一、問卷答案審核

### 審核流程

```
填答者提交 → 系統自動驗證 → 人工審核 → 通過/退回 → 鎖定版本
                  │                 │
                  ▼                 ▼
              自動產出          退回修正
            驗證報告           (附修改建議)
```

### 審核矩陣

| 填答分類 | 填答角色 | 一級審核 | 二級審核（SSOT 變更時） |
|---------|---------|---------|---------------------|
| 專案基本資訊 | PM | 架構師 | — |
| 系統架構 | 架構師 | PM + 資安 | 架構委員會 |
| 後端技術棧 | 後端主管 | 架構師 | — |
| 前端技術棧 | 前端主管 | 架構師 | — |
| 資料庫設計 | 後端主管 | 架構師 | — |
| API 設計 | 後端主管 | 架構師 | — |
| 資安與合規 | 資安主管 | 架構師 + PM | — |
| 部署與維運 | DevOps | 架構師 | — |
| AI 協作規範 | 架構師 | PM + 資安 | 架構委員會 |

### 審核標準

審核者需確認：

1. **完整性**：必填題是否全部回答
2. **一致性**：答案與其他分類是否矛盾
3. **合理性**：技術選型是否符合公司標準與專案需求
4. **信心度**：標記為 `unknown` 的項目是否有備註說明
5. **風險識別**：是否有已知風險未標記

### 審核結果

| 結果 | 說明 | 後續動作 |
|------|------|---------|
| ✅ 通過（Approved） | 答案完整且正確 | 鎖定版本，允許輸出 |
| ⚠️ 有條件通過 | 有 Warning 但無 Error | 可輸出但須追蹤改善 |
| 🔄 退回修正（Rejected） | 有錯誤或重大遺漏 | 退回填答者修正 |
| ❓ 需討論 | 需跨角色討論才能決定 | 安排技術會議 |

## 二、文件產出審核

### 自動產出文件的審核

| 產出文件 | 審核方式 | 審核者 |
|---------|---------|--------|
| `tech-stack.json` | PR Review | 架構師 |
| `ai-context.md` | PR Review | 架構師 |
| `readiness-report.md` | 自動產出 + 人工確認 | PM + 架構師 |
| `adr-draft.md` | 人工編輯後 PR Review | 架構師 |
| `cursor-rules/*.mdc` | PR Review | 架構師 + 開發者代表 |

### 審核重點

- 產出是否正確反映問卷答案
- 格式是否符合 Schema 定義
- 是否有遺漏的關鍵資訊
- 是否與 SSOT 文件一致
- AI context 是否足夠清晰供 AI 使用

## 三、AI 產出審核

### 審核清單

所有 AI 協作產出必須檢查：

- [ ] **功能正確性**：程式碼是否符合需求
- [ ] **風格一致性**：是否遵循專案的程式碼風格
- [ ] **安全性**：是否有安全漏洞（注入、洩漏）
- [ ] **無機敏資訊**：是否有硬編碼的密碼/金鑰
- [ ] **相容性**：是否與指定的技術棧版本相容
- [ ] **測試覆蓋**：是否有對應的測試
- [ ] **文件更新**：是否需要更新文件
- [ ] **SSOT 一致**：是否與 SSOT 文件一致

### AI 產出的特別關注點

| 關注項目 | 檢查方式 | 說明 |
|---------|---------|------|
| 幻覺(Hallucination) | 人工驗證 | AI 可能產出不存在的 API 或套件 |
| 過時資訊 | 版本比對 | AI 可能使用舊版語法或已棄用的 API |
| 隱含假設 | 邏輯檢查 | AI 可能假設不正確的執行環境 |
| 過度工程 | 設計審查 | AI 可能產出過度複雜的實現 |
| 不完整實現 | 功能測試 | AI 可能遺漏邊界情況 |

## 四、品質門檻（Quality Gates）

### Gate 1：填答完成門檻

- 所有必填題已回答
- 無 Error 等級的驗證規則違規
- Readiness Score ≥ 50

### Gate 2：可輸出門檻

- Gate 1 通過
- 至少一位審核者 Approved
- 無未解決的退回修正
- Readiness Score ≥ 70

### Gate 3：可進入 AI 協作門檻

- Gate 2 通過
- AI 協作規範分類完整填答
- AI 安全邊界已定義
- Context 管理策略已確認
- Readiness Score ≥ 70（AI 協作分類 ≥ 80）

### Gate 4：可發布 GitHub 門檻

- Gate 3 通過
- 所有產出文件已審核
- SSOT 標記已確認
- PR Template 已填寫完整
- AI 使用揭露已完成（若適用）
