# AI 使用治理政策 v1.0

> 本文件定義公司在專案開發中使用 AI 工具的治理政策、使用邊界與揭露規範。
> 此文件為 SSOT，任何 AI 使用相關決策以本文件為準。

---

## 一、AI 定位原則

### 核心定位

> **AI 為輔助起草者（Draft Assistant），不是事實來源（SSOT）。**

- AI 產出的所有內容，必須經由有責任的人工角色審核後，才能成為正式資產
- 真正的事實來源由 SSOT 文件承擔（如 OpenAPI Spec、Rules 定義、Error Code 定義）
- AI 不得直接修改 SSOT 文件，只能產出草稿供審核

### 使用層級定義

| 層級 | 名稱 | 說明 | 審核要求 |
|------|------|------|---------|
| L1 | 探索輔助 | 用 AI 做技術調查、方案比較、概念驗證 | 無需審核 |
| L2 | 草稿產出 | 用 AI 產出程式碼草稿、文件草稿 | 需人工 Review |
| L3 | 協作開發 | AI 直接參與開發流程（如 Cursor Agent） | 需 Senior Review + PR |
| L4 | 自動化流程 | AI 觸發自動化操作（CI/CD、Issue 處理） | 需架構師核准 |

## 二、允許與禁止範圍

### 允許範圍（Allowed）

| 範圍 | 條件 | 層級 |
|------|------|------|
| 程式碼生成 | 須經 Code Review | L2-L3 |
| 單元測試撰寫 | 須驗證測試是否有效 | L2-L3 |
| 文件撰寫 | 須經內容審核 | L2 |
| 程式碼重構 | 須有對應測試覆蓋 | L3 |
| API 設計建議 | 須與架構師確認 | L1-L2 |
| 程式碼審查輔助 | 輔助工具，非最終決策 | L1 |
| 技術方案探索 | 結論需人工驗證 | L1 |
| Bug 修復建議 | 修復方案需人工確認 | L2 |

### 禁止範圍（紅線 — Red Lines）

| 禁止事項 | 原因 | 嚴重度 |
|---------|------|--------|
| 直接部署到正式環境 | 安全風險不可控 | 🔴 嚴格禁止 |
| 正式環境資料庫異動 | 資料安全 | 🔴 嚴格禁止 |
| 處理密碼/金鑰/憑證 | 機敏資訊保護 | 🔴 嚴格禁止 |
| Prompt 中包含個資(PII) | 個資法合規 | 🔴 嚴格禁止 |
| 未經審核直接合併 AI 產出 | 品質管控 | 🔴 嚴格禁止 |
| 讓 AI 成為規則來源 | SSOT 原則 | 🟡 禁止 |
| 刪除正式資料 | 不可逆操作 | 🔴 嚴格禁止 |
| 修改安全性組態 | 需人工操作 | 🟡 禁止 |

## 三、AI 使用揭露規範

### PR 揭露要求

所有包含 AI 協作產出的 Pull Request，必須在 PR 描述中揭露：

```markdown
## AI 使用揭露
- [ ] 本 PR 包含 AI 協作產出
- 使用工具：（如 Cursor / Copilot / Claude）
- AI 參與範圍：（如 程式碼生成 / 測試撰寫 / 重構建議）
- 人工審核範圍：（如 所有檔案 / 僅核心邏輯 / 測試案例）
- AI 產出佔比估計：（如 30% / 50% / 70%）
```

### Commit 揭露建議

AI 協作產生的 commit，建議在 commit message 中標記：

```
feat(api): add user endpoint [ai-assisted]
```

或使用 Git Trailer：

```
Co-authored-by: AI Assistant (Cursor/Claude)
```

## 四、安全邊界

### 資料接觸限制

| 資料等級 | AI 可接觸 | 條件 |
|---------|----------|------|
| 公開資料 | ✅ 是 | 無限制 |
| 內部資料 | ✅ 是 | 不得外傳至非公司核准的 AI 服務 |
| 機密資料 | ⚠️ 有限 | 僅結構/Schema，不含實際資料 |
| 高度機密 | ❌ 否 | 禁止在 AI prompt 中出現 |

### 模型使用限制

- 僅使用公司核准的 AI 模型清單中的模型
- 禁止將公司程式碼上傳至未經核准的第三方 AI 服務
- 使用雲端 AI 時，需確認資料不會被用於模型訓練

### 核准模型清單（範例）

| 模型 | 用途 | 核准狀態 |
|------|------|---------|
| Claude (Anthropic) | 程式碼生成、文件、審查 | ✅ 核准 |
| GPT-4o (OpenAI) | 通用輔助 | ✅ 核准 |
| Azure OpenAI | 企業級 API 呼叫 | ✅ 核准 |
| GitHub Copilot | IDE 內程式碼輔助 | ✅ 核准 |
| 開源本地模型 | 機密專案使用 | ⚠️ 需個案評估 |

## 五、稽核與追蹤

### 稽核項目

- AI 使用工具與頻率
- AI 產出佔比趨勢
- AI 產出的 Bug 率
- Code Review 通過率
- 安全性問題發現數

### 追蹤機制

- 每月統計 AI 使用揭露的 PR 數量
- 每季檢視 AI 協作品質指標
- 安全事件即時通報
- 年度 AI 使用政策檢討

## 六、政策維護

| 項目 | 說明 |
|------|------|
| 負責人 | 架構委員會 + 資安主管 |
| 審核週期 | 每季檢視一次 |
| 變更流程 | PR → 架構師 + 資安審核 → 合併 |
| 生效條件 | 合併至 main 分支即生效 |
