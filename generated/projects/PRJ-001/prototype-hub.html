PRJ-001 Prototype Hub

這一頁是 `S1-S6` prototype 的總入口，方便依照階段順序操作、匯入前階段 JSON、再匯出下一階段資料。
建議從 `S1` 開始，逐階匯出 `project-context.json` 或該階段 handoff 檔，再帶進下一階段。
從 S1 開始 查看最終輸出

建議操作順序

  • `S1 Discovery`：建立專案基本資料，匯出 `project-context.json` 或 `S2 decision list`。
  • `S2 Planning`：定義架構方向與 AI 邊界，回存更新版 `project-context.json`。
  • `S3 Design`：由各技術角色補齊可實作的工程規格，匯出 `S4 validation checklist`。
  • `S4 Validation`：自動檢查 + 人工處理，匯出 `S5 review packet`。
  • `S5 Review`：架構師與 PM 簽核，鎖定版本，匯出 `S6 output handoff`。
  • `S6 Output`：生成正式輸出檔，供 repo 與 AI 工具使用。

各階段入口

S1 Discovery

PM
確認專案基本資料、範圍、敏感度、前置文件與 AI 邊界基線。
  • 主要輸入：專案背景、需求摘要、公司方法資料
  • 主要輸出：S1 answers / `project-context.json` / S2 decision list
目前狀態：已建立，可作為整個流程起點。
建議下一步：完成 S1 後，直接帶 `project-context.json` 進 S2。
前往 S1

S2 Planning

Architect
定義架構方向、平台選型、AI 協作邊界與關鍵決策。
  • 主要輸入：S1 匯出的 `project-context.json`
  • 主要輸出：S2 context / S3 decision list / 更新版 `project-context.json`
目前狀態：已建立，可先做架構方向與 AI 邊界決定。
建議下一步：確認核心題後，把輸出帶進 S3 做分角色設計。
前往 S2

S3 Design

Tech Leads
補齊 backend / frontend / database / api / security / deployment 的實作規格。
  • 主要輸入：S2 context / `project-context.json`
  • 主要輸出：S3 context / S4 validation checklist / 更新版 `project-context.json`
目前狀態：已建立，適合各技術角色開始細化規格。
建議下一步：補齊六大分類後，進入 S4 做驗證與問題收斂。
前往 S3

S4 Validation

System + Architect
自動規則檢查、Blocker / Warning 分流、人工處理與 Gate 3 判定。
  • 主要輸入：S3 context / `project-context.json`
  • 主要輸出：validation report / S5 review packet / 更新版 `project-context.json`
目前狀態：已建立，可用來做 Gate 3 前的自動檢查與人工處理。
建議下一步：把 blocker 清空、warning 收斂後，帶進 S5 審核。
前往 S4

S5 Review

Architect + PM
正式審核、退回修正、簽核與版本鎖定。
  • 主要輸入：S4 validation report / S5 review packet / `project-context.json`
  • 主要輸出：approved review packet / S6 handoff / 更新版 `project-context.json`
目前狀態：已建立，可作為正式簽核與版本鎖定台。
建議下一步：Architect 與 PM 都完成決定後，直接進 S6 產出正式文件。
前往 S5

S6 Output

System
生成可放入 repo 與 AI 工具使用的正式輸出文件。
  • 主要輸入：S5 approved review packet / S6 handoff / `project-context.json`
  • 主要輸出：`tech-stack.json`、`ai-context.md`、`readiness-report.md`、`adr-draft.md`、`cursor-rules-draft.mdc`
目前狀態：已建立，可直接生成正式輸出資產。
建議下一步：產出後即可準備進入發布流程或建立 PR。
前往 S6
